# EcoTrack Governance

This document describes the governance model for the EcoTrack project.

## Roles

### 1. Maintainers
Maintainers have push access to the repository and are responsible for:
- Reviewing and merging pull requests.
- Triaging issues and guiding project architecture.
- Releasing new versions and managing deployments.

Current Maintainer(s):
- [@Abb2907](https://github.com/Abb2907)

### 2. Contributors
Contributors are community members who submit issues, propose features, or open pull requests. All contributors are expected to follow the [Code of Conduct](./CODE_OF_CONDUCT.md) and [Contributing Guidelines](./CONTRIBUTING.md).

## Decision Making
Major changes to the application architecture, database schemas, or infrastructure require consensus among Maintainers, usually tracked via GitHub Issues or Architecture Decision Records (ADRs).

## Becoming a Maintainer
Active contributors who consistently demonstrate a strong understanding of the codebase and project goals may be invited to become Maintainers.
